The Kind Programming Language

The Kind Programming Language is a programming language built on top of the BEAM virtual machine (home of Erlang and Elixir). It's created to try out a few fanciful ideas in type system design and static analysis. One of which is the idea that all functions are considered total and any input a function isn't defined for leads to a compile time error.

Features

  • Algebraic Data Types (done)
  • Pattern Matching (partially done)
  • Erlang Interop (partially done)
  • Module System (done)
  • Type Classes (pending)
  • Numbers (pending)
  • Strings (pending)

Examples

Main function always returning `False`

def alwaysFalse -> False def main -> alwaysFalse()

Main function always returning `True`

def alwaysTrue -> True def main -> alwaysTrue()

How to Use

The Kind language is still in its infancy, and while I'd love for there to be a nifty little executable that can compile your source into binaries, this isn't the case (yet). For playing around, feel free to edit the examples on this website to test out ideas. If that doesn't satisfy your curiosity, then check out the Kind source code on Github and play around with the kind module.
